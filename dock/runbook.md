## 0. 共通の第一声

> 「画面を見せてもらっていいですか？」

**指示する前に現状を見る**。同じ症状でも原因が違うことが多い。
口頭だけで直そうとしない。

参加者の状態を確認する3点セット:

- **どのページ**にいるか（URL のハッシュ `#/...`）
- **OS**（Windows / Mac）と **ブラウザ**（Chrome / Edge / Safari）
- **何をクリックしたら何が出たか**（エラーメッセージの文面を見る）

## 1. 教材サイトそのもののトラブル

### 1-1. ページが真っ白 / エラー画面が出る

- **症状**: オレンジ色の「画面の表示でエラーが発生しました」画面
- **切り分け**:
  1. まず **「再読み込み」** ボタン（ほぼこれで直る）
  2. 直らない場合は **「進捗をリセットして再読み込み」**（localStorage 破損時の脱出口）
  3. それでも直らない → ブラウザの DevTools (F12) → Console タブを見せてもらう
- **外部要因**: 校内ネットワークが CDN（cdn.jsdelivr 等）を弾いている可能性。
  テザリングに切り替える。

### 1-2. GitHub Pages が 404

- **症状**: `https://<user>.github.io/yuru-kouza-web/` にアクセスして 404
- **切り分け**:
  1. リポジトリの **Settings → Pages** で Source が **「GitHub Actions」** になっているか
  2. **Actions** タブで最新の `Deploy to GitHub Pages` が緑（成功）になっているか
  3. 失敗している場合はログを展開 → `pnpm install` か `pnpm build` のどちらで落ちたか
- **よくある原因**:
  - `pnpm-lock.yaml` が壊れている → ローカルで `pnpm install` → 再 push
  - `VITE_BASE_PATH` が workflow からしか渡らない → `main` ブランチに直接 push しているかを確認

### 1-3. 進捗チェックが保存されない

- **症状**: ✓ を付けても、別ページに行って戻ると外れている
- **切り分け**:
  1. **プライベートブラウジング** で開いていないか（localStorage が拒否される）
  2. ブラウザ設定で「すべての Cookie とサイトデータをブロック」になっていないか
- **対処**: 通常ウィンドウで開き直す。今日だけの対応なら
  「プライベートモードだとタブ閉じたら消えるよ」と案内。

### 1-4. OS 判定がおかしい

- **症状**: Mac なのに Windows タブが表示される / 逆
- **対処**: 右側「手動切替」ボタンで切り替えてもらう。
  一度切り替えれば全ページで同期される（NFR 的仕様）。

## 2. GitHub アカウント・認証系

### 2-1. アカウント作成でメール認証が来ない

- **5分待つ**（GitHub 側の遅延）
- スパムフォルダ / 迷惑メールを確認
- それでも来ない → **運営に連絡**（イベント用捨てアドを貸す最終手段）

### 2-2. 「Please verify your account」ループ

- GitHub の bot 判定に引っかかっている
- アバター画像を設定 / プロフィールを少し埋める / 数分待つ
- テザリング等で IP を変えると通ることがある

### 2-3. 2FA が必須になって詰む

- **今日はスキップ**してもらう（イベント中は MFA 設定しない方針）
- 帰宅後に設定する旨をメモ

### 2-4. OAuth 承認画面が出ない / VSCode で GitHub ログインが無限ループ

- **切り分け**:
  1. ブラウザの **ポップアップブロック** 解除
  2. 一度 **VSCode を再起動**
  3. VSCode 左下のアカウントアイコンから **サインアウト → サインイン**
- それでもダメなら、VSCode の **設定 → 拡張機能 → GitHub** で
  Device Code フローに切り替える

### 2-5. Organization の承認が要求される

- イベント会場のネットワークや端末の都合で組織所有のリポジトリにアクセスできない場合がある
- **個人アカウントのリポジトリ**として作り直してもらう

## 3. VSCode・Git 系

### 3-1. Clone ボタンが押せない / 候補が出ない

- GitHub 拡張が未ログイン → セクション 2-4
- 拡張機能が入っていない → `ms-vscode.github-extensions` を入れる
- 代替案: `<>` ボタン緑の「Code」から URL コピー → コマンドパレットで Clone

### 3-2. Clone 先フォルダが選べない

- フォルダを **新規作成** するか、既存の空フォルダを選んでもらう
- OneDrive / iCloud 同期フォルダは **避ける**（権限と同期衝突で詰みやすい）

### 3-3. ソース管理タブに「＋」が出ない

- ファイルを保存していない → Ctrl/Cmd + S
- `.gitignore` で除外されている → 今回は発生しないはず

### 3-4. Push で認証ダイアログが無限に出る

- セクション 2-4 と同じ
- **Personal Access Token** を手動で発行して貼り付ける手は **最終手段**
  （時間を食う。今日は OAuth で通す方を頑張る）

### 3-5. Push 成功したが GitHub に反映されない

- **F5 でページ更新**（GitHub 側キャッシュ）
- 違うブランチに push している → `main` になっているか VSCode 左下で確認
- 別リポジトリにコミットしている → `origin` の URL を確認

## 4. 「特別な README」が表示されない

参加者が一番ハマる場所。

### チェック順（ここだけは暗記）

1. **リポジトリ名 == ユーザー名** か？（大文字小文字含めて完全一致）
2. リポジトリが **Public** か？（Private だとプロフィールに出ない）
3. `README.md` に **1文字でも中身がある** か？（空ファイルだと出ない）
4. ブラウザを **F5** （GitHub のプロフィールキャッシュ）

3つ揃っていてもダメなら:

- ファイル名が `Readme.md` や `readme.md` になっていないか（`README.md` 推奨、
  GitHub は大小区別しないが統一のため）
- ブランチが `main` か（`master` や他ブランチに書いてないか）

## 5. エスカレーション基準

### 自分で抱え込まず運営に投げるライン

- **30 分たっても解決しない**
- **同じ症状が 3 人以上**（環境起因 or 教材の不備の可能性）
- **アカウント作成で CAPTCHA が通らない** 等、技術で解決不可能な外部要因
- **参加者が投げ出しかけている**（精神的なケアが優先）

### 記録してほしいこと

Runbook を改善するため、以下を Slack / 共有ドキュメントに残す:

- 症状（一言）
- 環境（OS / ブラウザ）
- 最終的にどう解決したか / どう迂回したか
