# 要件定義書

## 1. プロジェクト概要

### 1.1 目的
4/26 事前講座 Web班の教材Webサイト。参加者が**手元のスマホで教材を参照しながらPCで作業できる環境**を提供し、全員が「GitHubプロフィールに自己紹介READMEを表示する」成功体験を得られるようにする。

### 1.2 スコープ
- **In Scope**
  - 4/26 Web班の手順書（共通モジュール含む）
  - 当日進捗を保存・再開する機能
  - Windows / Mac 固有手順の切替表示
  - 早く終わった参加者向けの発展コンテンツ（予習相当）
- **Out of Scope**（本SDD対象外。ただし4/29教材は**同一リポジトリ内の別パスで後日追加**予定）
  - Unity班の教材（別資料）
  - 4/29ハッカソン本番用の教材（同リポジトリに後日追加。本サイトからは予習リンクのみ提供）
  - チーム編成・アイディアソン支援
  - SSH設定手順（挫折防止のため本番でも未実施）

### 1.3 成功基準
- **定性**: 参加者が迷わず自己紹介READMEを公開できる
- **定量**:
  - 完走率 90% 以上（GitHub プロフィールに README 表示まで到達）
  - 教材起因の離脱 10% 以下

## 2. ユーザーストーリー

### US-1: 初心者参加者
- **UC-1.1** スマホで教材を見ながらPCで作業したい → レスポンシブ必須
- **UC-1.2** 自己紹介テンプレを手打ちせずに貼り付けたい → コピーボタン付きテンプレブロック
- **UC-1.3** 進捗を残してトイレ休憩や迷子から復帰したい → 進捗チェックボックス + `localStorage`
- **UC-1.4** 自分のOSの手順だけ見たい → OS切替UI

### US-2: 早く終わった参加者
- **UC-2.1** 追加で学べる内容が欲しい → 発展コンテンツ
- **UC-2.2** 4/29の予習をしたい → HTML/CSS/JS基礎、GitHub Pages概要

### US-3: メンター
- **UC-3.1** つまずきそうな箇所を把握したい → 各ページに「よくあるつまずき」セクション
- **UC-3.2** 参加者画面をのぞいて進捗を確認できる → 進捗チェックは画面に常時表示

### US-4: 運営（執筆者・保守者）
- **UC-4.1** Markdownだけで編集できる → MDベース、Vueコンポーネントは名前を覚えるだけで使える
- **UC-4.2** 無料でホストしたい → 静的サイト + GitHub Pages
- **UC-4.3** 本番直前の差し替えを安全にしたい → PR → 自動デプロイ

## 3. 機能要件（FR）

### FR-1: 教材閲覧
システムは参加者が階層構造の教材を閲覧できる機能を提供する。
- FR-1.1: トップページに全体アナウンス（4/26の位置づけと4/29との関係）を表示
- FR-1.2: サイドバーナビゲーション（共通 → Web班 → 発展）
- FR-1.3: 各ページ末に「次のステップ」への明示的リンク
- FR-1.4: ローカル全文検索（VitePress標準機能）

### FR-2: レスポンシブデザイン
システムはスマートフォン（最小幅320px〜）・タブレット・PCで読みやすいUIを提供する。
- FR-2.1: ブレイクポイント: 〜767px（モバイル）、768〜1023px（タブレット）、1024px〜（PC）
- FR-2.2: モバイル時はサイドバーをハンバーガーメニューに格納
- FR-2.3: コードブロックは横スクロール可能、文字が潰れない最小サイズを確保
- FR-2.4: タップターゲット 44px 以上

### FR-3: コピペ用Markdownテンプレート
システムは自己紹介READMEテンプレートをワンクリックでコピーできる機能を提供する。
- FR-3.1: コピーボタン付きのコードブロックコンポーネント
- FR-3.2: 書き換え対象のプレースホルダー（例: `<あなたの名前>`）を視覚的に強調
- FR-3.3: コピー成功時のフィードバック（トースト、2秒程度）
- FR-3.4: クリップボードAPI不可環境（古いブラウザ等）でも手動選択は可能

### FR-4: 進捗トラッカー
システムは参加者が各手順の完了状態を記録・永続化し、再訪時に復元できる機能を提供する。
- FR-4.1: 各手順の末尾にチェックボックスを設置
- FR-4.2: チェック状態を `localStorage` に保存
- FR-4.3: ページ再訪時に状態を復元
- FR-4.4: サイドバーで各セクションの完了率を視覚化（✓ マーク or 「3/5」表記）
- FR-4.5: 全進捗をリセットするボタンを設置（**P0: 当日の復旧手段として必須**。ストレージ破損・状態不整合から復帰できる唯一の手段）
- FR-4.6: `localStorage` 非対応/不許可時も閲覧は成立する（機能劣化モード）

### FR-5: OS切替
システムはWindows / MacでUI上の固有手順を切り替えて表示できる機能を提供する。
- FR-5.1: 初回アクセス時に `navigator.userAgentData.platform`（新API）を優先、失敗時に `navigator.platform` で自動判定
- FR-5.2: タブUI（Windows / Mac）で手動切替可能
- FR-5.3: 選択状態を `localStorage` に保存、ページ間で一貫
- FR-5.4: OS固有の差分がある箇所にのみタブを設置（全体を分岐させない）
- FR-5.5: 同一ページ内の複数のタブは選択状態が同期する

### FR-6: 発展コンテンツ
システムは早く終わった参加者向けに発展的な内容を別セクションで提供する。
- FR-6.1: 共通発展: SSH接続、GitHub Actions概要
- FR-6.2: Web班向け発展: HTML/CSS基礎、JavaScript基礎、GitHub Pages概要、CSSフレームワーク紹介
- FR-6.3: 「おまけ」と明示し、メインフローと混同させない（サイドバーでも区分）

## 4. 非機能要件（NFR）

### NFR-1: パフォーマンス
- 初回ロード LCP < 2.5秒（3G/4G想定）
- ビルド時に静的生成、配信は純粋な静的ファイル

### NFR-2: 可用性
- GitHub Pages等の静的ホスティングで配信
- 認証不要、誰でも閲覧可能

### NFR-3: 保守性
- コンテンツは Markdown、非エンジニアでも編集可能
- 構造変更はサイドバー定義（`config.ts`）のみで完結
- コンポーネントは再利用可能な単位（進捗/OS/コピー）に分離

### NFR-4: アクセシビリティ
- WCAG 2.1 AA 準拠を目標
- キーボードのみで全操作可能
- カラーコントラスト比 4.5:1 以上

### NFR-5: ブラウザ対応
- Chrome / Edge / Safari / Firefox の最新2バージョン
- iOS Safari / Android Chrome（スマホ必須）

### NFR-6: セキュリティ・プライバシー
- サーバー処理なし（完全静的）
- `localStorage` に個人特定情報を保存しない（進捗状態とOS設定のみ）
- 外部トラッキングは載せない

### NFR-7: 多言語
- 日本語のみ（国際化は不要）

## 6. 優先順位

| 優先度 | 項目 |
|---|---|
| **P0（必須）** | FR-1, FR-2, FR-3, **FR-4**, FR-5, NFR-1, NFR-3, NFR-5 |
| **P1（重要）** | NFR-4 |
| **P2（あれば良い）** | FR-6, ローカル全文検索の高度化 |

P0 が欠けると当日の教材として機能しない。P1 は UX を明確に底上げする。P2 は時間があれば着手する。

**2026/04/18 レビューでの変更**: FR-4（進捗トラッカー全体）を P1 → **P0** に昇格。理由: FR-4.5（リセット）は当日の復旧手段として必須であり、リセットが存在するためには本体機能も確実に動作する必要があるため、機能全体を P0 として扱う。
