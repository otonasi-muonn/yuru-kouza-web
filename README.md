# ゆるハッカソン事前講座 教材サイト SDD仕様書

4/26（日）事前講座 Web班向けの教材Webサイトの仕様一式。

## 確定事項（2026/04/18 レビュー後）

| 項目 | 決定 | 補足 |
|---|---|---|
| 技術スタック | **VitePress + TypeScript** | 確定 |
| ホスティング | **GitHub Pages** | 4/29本番で参加者が使うツールと一致＝**ドッグフーディング**として教材性がある |
| リポジトリ構成 | **4/26と4/29を同一リポジトリ**でホスト | 本SDDは4/26分のみカバー。4/29分は後日別パス（例: `/hackathon/`）で追加 |
| 進捗リセット機能 | **P0に昇格** | 当日の復旧手段として必須 |
| スクリーンショット（T-17） | 運営で担当割り | Win/Mac両方が必要 |

## 構成

| ファイル | 役割 |
|---|---|
| `requirements.md` | 要件定義（ユーザーストーリー / FR / NFR / 優先度） |
| `design.md` | 技術設計（アーキテクチャ / コンポーネント / データモデル） |
| `tasks.md` | タスク分解（マイルストーン / 見積 / リスク） |
| `content-outline.md` | 教材コンテンツの構成と各ページの狙い |

## 開発フロー（SDD）

1. `requirements.md` をレビューし、合意する
2. `design.md` の技術選定と論点への判断を確定する
3. `tasks.md` の優先度（P0 / P1 / P2）に従って順に実装
4. `content-outline.md` を見ながら Markdown を書く

## 基本情報

- 対象イベント: 4/29（水・祝）ゆるハッカソン本番の事前講座（4/26開催）
- 対象部門: Web班（＋共通モジュール）
- 本SDDの対象外: Unity班の教材、4/29本番用教材
  - ※ 4/29本番用コンテンツは**同リポジトリ内の別パス**で後日追加する前提
- 成功の定義: 参加者が自分の GitHub プロフィールページに自己紹介 README が表示された状態
- スコープ制約: SSH設定なし・HTML/CSS/デプロイは本番側（事前講座では扱わない）
